## Публикация сайта

Перед публикацией убедитесь, что все изменения сохранены в Git. Выполните следующие действия:

- Проверьте, нет ли несохраненных файлов: `git status`.
- Добавьте все изменения: `git add .`.
- Запишите изменения: `git commit -m "Документация готова к публикации"`.
- Отправьте изменения на GitHub: `git push`.
- Выполните команду: `python -m mkdocs gh-deploy --clean`.

### Настройка GitHub Pages

Сайт собран, но GitHub не показывает его по адресу и показывает ошибку 404. Выполните следующие действия:

- Откройте в браузере страницу вашего репозитория.
- Нажмите на вкладку Settings, в правом верхнем углу.
- Нажмите Pages в левом меню.
- В разделе "Branch" выберите:
  1. В выпадающем списке: gh-pages (ветка публикации сайта).
  2. Рядом оставьте: /(root).
  3. Нажмите кнопку Save.
- Подождите 1-2 минуты, пока GitHub активирует сайт.
